:abc: 软件需求管理过程：

+ 需求获取：是软件开发项目最关键的一个输入
+ 需求分析：也称为需求建模，是为最终用户所看到的系统建立的一个概念模型
+ 需求规格编写：为了使用户和开发者双方对软件的初始规定有一个共同的理解
+ 需求验证：分析需求的正确性和可行性；
+ 需求变更：是软件项目的一个突出的特定，也是软件项目最为普遍的一个特点



:accept: 时间管理的金三角

+ 范围， 时间， 成本

:it: IT项目分解步骤

+ 分解要素
+ 分解标准
+ 明确责任
+ 确定成果
+ 验证分解



:dancers: IT 项目管理时间和成本的管制要点:

+ 时间：

1. 项目要义，指确定一些特定的工作；
2. 项目排序，明确项目内各个部分的相互联系性；
3. 项目时间估计；
4. 进度计划编制；
5. 进度控制

+ 成本：

1. 资源计划，决定实施项目活动需要使用的寿命资源及每种资源的用量和使用时间；
2. 成本估算，估计完成项目所需资源成本的近似值；
3. 成本预算，将整体成本估算分配到各个单位工作，建立一个衡量绩效的基准计划
4. 成本控制，控制项目预算的变化；



:japanese_goblin: IT项目管理怎么管理团队

​	:1st_place_medal: 指定良好的规章制度，一个强劲的管理团队是建立在一套好的规章制度上的；

​	:2nd_place_medal: 制定明确的共同目标，团队中不同角色由于看问题角度不同，对项目的目标的期望值也会不同，故一个共同的奋斗目标是必要的





:3rd_place_medal: IT 项目管理变更

	1. 提交和接收变更申请
 	2. 审核和记录变更申请
 	3. 确定变更申请的可行性
 	4. 批准变更申请
 	5. 实施和结束变更申请



:scream_cat: 概要设计：就是设计软件的结构，包括组成模块，模块的功能，模块间的调用关系；

同时设计该项目应用系统的总体数据结构和数据库结构；即应用系统要存储什么数据，这些数据是什么结构，他们之间有什么关系；

:cake: 概要设计是详细设计的基础，必须在详细设计之前完成，概要设计经过复查之后在开始进行详细设计，概要设计必须完成概要设计文档，包括系统设计文档，

:dancer: 概要设计里的功能应该重点在功能描述上，对需求的解释和整合，整体划分模块和对各个功能模块进行详细的图文说明；

:v:  概要设计里的数据库设计应该重点在描述数据关系上，说明数据的来龙去脉，设计目的和原因





:b: 详细设计是对概要设计的一个细化，就是详细设计每个模块的实现算法和局部结构，在详细设计阶段，主要是通过需求分析的结果，设计出满足用户需求的嵌入式系统产品；

详细设计要完成详细设计文档；

详细设计里的数据库设计应该是一份完善的数据结构文档，就是一个包括类型，命名，精度，字段说明，表说明等内容的数据文档；
